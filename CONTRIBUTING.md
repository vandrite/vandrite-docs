# Contributing to Vandrite Documentation

Thank you for your interest in contributing to the **Vandrite Documentation**! We welcome contributions from the community to help make our documentation better for everyone.

## ü§ù How to Contribute

### Reporting Issues

If you find a bug, a typo, or unclear content, please open an issue in our repository. When reporting an issue, please include:

- A clear and descriptive title.
- A link to the page where the issue exists.
- A screenshot (if applicable).
- A description of what is wrong and how it should be fixed.

### Submitting Changes

1. **Fork the repository** to your own GitHub account.
2. **Clone the project** to your local machine.
3. **Create a new branch** for your changes:
   ```bash
   git checkout -b my-feature-branch
   ```
4. **Make your changes** to the documentation.
5. **Preview your changes** locally:
   ```bash
   npm run dev
   ```
6. **Commit your changes**:
   ```bash
   git commit -m "docs: fix typo in installation guide"
   ```
7. **Push to your branch**:
   ```bash
   git push origin my-feature-branch
   ```
8. **Open a Pull Request** (PR) from your forked repository to the main Vandrite repository.

## üìù Style Guide

- **Tone:** Professional, friendly, and concise.
- **Formatting:** Use Markdown. Ensure headings are properly nested.
- **Code:** Use code blocks with language identifiers (e.g., `bash`, `typescript`).
- **Links:** Use relative links for internal navigation and absolute links for external resources.
